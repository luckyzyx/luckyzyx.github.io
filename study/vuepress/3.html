# 基本配置

config.js的主要配置包括网站的标题、描述等基本信息，以及主题的配置。这里简单的列举一下常用配置。

# 网站信息

module.exports = {
    title: 'work', 
    description: '个人博客',
    head: [
        ['link', { rel: 'icon', href: '/img/logo.ico' }],
        ['link', { rel: 'manifest', href: '/manifest.json' }],
    ]
}

# 主题配置

module.exports = {
       themeConfig: {
           //nav导航栏
        nav: [
            { text: '主页', link: '/' },
            { text: '博客',
              items: [
                { text: '一', link: '/blog/1' },
                { text: '二', link: '/blog/2' },
                { text: '三', link: '/blog/3' }
              ] 
            },
            { text: '关于', link: '/about/' },
            { text: 'Github', link: 'https://github.com' },
        ],
        sidebar: {//sidebar侧边栏
            '/blog/': [
                        "",//默认README.md
                        "1", 
                        "2",
                        "3",
                         ],
                "/demo/":[
                        "",
                        "1",
                        ],

            },
        sidebarDepth: 2,
        lastUpdated: 'Last Updated', //上次更新时间，可更改
    },
}

nav：导航栏配置，主要用于配置导航栏的链接，例如主页的link为"/"，默认是根目录下的README.md.

"/blog/"链接到根目录docs下的blog文件夹下的README.md文件。

sidebar：侧边栏配置，可以省略.md后缀拓展名，同时以/结尾的路径将会被视为 */README.md。

'/blog/'、'/demo/'通过路由方式将每个页面的标题抽取出来显示。

"/blog/"是指根目录下blog文件夹中的路由，每个路由链接都要有README.md。

目录结构如下：

├─── docs
├── README.md
└── blog
│   └── README.md
└── demo
   └── README.md
